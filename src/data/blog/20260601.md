# 插件开发学习笔记

本文档用于持续整理当前对话里提到的浏览器插件开发知识点。

- 对话时先回答当前问题，再在回答后自动整理新增知识点。
- 如果只是闲聊或问题和插件开发无关，不强行更新本文档。
- 笔记按主题归类，不按提问时间堆叠。

## 1. 浏览器插件开发流程

浏览器插件可以先理解成几块代码协作：

- `manifest.json`：插件总配置，声明名称、权限、入口、背景脚本、内容脚本等。
- `sidepanel / popup`：插件自己的界面，负责按钮、输入框、配置和状态展示。
- `background / service worker`：插件后台，负责调度、状态、消息处理。
- `content script`：注入网页里的脚本，负责读取页面、找元素、填表单、点按钮。

一个典型调用链：

1. 用户在侧边栏点击按钮。
2. 侧边栏脚本发消息给后台。
3. 后台决定在哪个标签页执行操作。
4. 内容脚本在目标网页里查找 DOM 元素并操作页面。

## 2. 常用 IDE 和工具

最常用的组合：

- `VS Code`：写 `HTML / CSS / JavaScript`，全局搜索，看目录结构。
- `Chrome / Chromium`：加载未打包扩展，调试插件页面和内容脚本。
- `Chrome DevTools`：打断点、看变量、查网络请求、查看 DOM、看 storage。
- `Node.js`：很多项目会用它跑测试、脚本或构建工具。

插件开发不一定要先上框架。原生 `HTML + CSS + JavaScript + manifest.json` 就能开发很多插件。

后面需要工程化时，再考虑：

- `WXT`
- `Plasmo`

当前仓库更偏原生扩展结构，不是典型前端脚手架项目。

## 3. Chrome DevTools

`Chrome DevTools` 是 Chrome 自带的开发者调试工具。

它在插件开发里基本承担这些角色：

- 调试器
- 对象查看器
- DOM 检查器
- 网络面板
- 存储查看器

最值得先学的面板：

- `Sources`：最接近传统 IDE 断点调试。
- `Console`：临时执行表达式、查看对象。
- `Network`：看请求、响应、Headers、Cookie、耗时。
- `Elements`：查看 DOM 树和样式。
- `Application`：查看 storage、cookies 等。

### 3.1 像 Python 一样调试插件

插件也可以打断点、单步执行、看变量、看调用栈。

常见的三个调试上下文：

- `sidepanel`：调试插件自己的界面逻辑。
- `content script`：调试注入网页中的脚本。
- `background / service worker`：调试后台逻辑。

和 Python 调试最大的区别是：插件不是单进程思维，而是多个运行上下文。

调试时先问：

**这段代码到底跑在哪一层？**

### 3.2 `debugger`

前端里可以直接写：

```js
debugger;
```

代码运行到这里会自动停住，效果类似手动打断点。

### 3.3 DevTools 里的 badge

DevTools `Elements` 面板里的 `badge`，不是 Chrome 扩展图标上的角标。

在这里，`badge` 指贴在 DOM 元素旁边的小提示标签，例如：

- `flex`
- `grid`
- `media`
- `scroll`
- `container`

`Badge settings` 的意思是：控制这些“小标签”要不要显示。

例如某个元素用了：

```css
display: flex;
```

它在 DOM 树旁边可能会出现 `flex` badge。点这个 badge，DevTools 可以显示 flex 布局的可视化辅助线。

## 4. DOM

`DOM` 是 `Document Object Model` 的缩写。

- `Document`：文档，这里指网页文档。
- `Object`：对象，表示浏览器会把网页内容转换成对象。
- `Model`：模型，表示它是一套组织和操作这些对象的方式。

DOM 树可以理解成：

**浏览器把 HTML 页面解析后，在内存中组织出来的一棵对象树。**

例如：

```html
<body>
  <div id="app">
    <h1>Hello</h1>
    <button>点我</button>
  </div>
</body>
```

浏览器大致会理解成：

```text
body
└── div#app
    ├── h1
    │   └── "Hello"
    └── button
        └── "点我"
```

JavaScript 操作网页，本质上就是在读写这棵 DOM 树。

## 5. 常见 HTML 标签

### 5.1 `<div>`

`<div>` 是通用块级容器。

它本身没有具体业务语义，主要作用是：

- 把一块内容包起来。
- 方便布局。
- 方便加样式。
- 方便给 JavaScript 查找和操作。

例子：

```html
<div id="card">
  <h1>标题</h1>
  <p>这是一段内容</p>
</div>
```

可以把 `div` 理解成一个盒子，里面装着标题和正文。

### 5.2 `<li>`

`<li>` 是列表项，通常可以理解成 `list item`。

它一般配合下面两种标签一起用：

- `<ul>`：无序列表，通常显示为圆点列表。
- `<ol>`：有序列表，通常显示为数字列表。

无序列表示例：

```html
<ul>
  <li>苹果</li>
  <li>香蕉</li>
  <li>橘子</li>
</ul>
```

有序列表示例：

```html
<ol>
  <li>打开浏览器</li>
  <li>进入扩展页</li>
  <li>加载插件</li>
</ol>
```

`div` 和 `li` 的区别：

- `div`：通用容器，什么内容都能包。
- `li`：列表项，表示列表中的一条。

### 5.3 `<style>`

`<style>` 标签表示这里面放的是 CSS 样式规则。

```html
<style>
  .title {
    color: red;
  }
</style>
```

意思是给页面补充一段样式。

## 6. CSS 选择器

CSS 选择器是一种“定位元素”的语法。

它有两个主要用途：

- 在 CSS 中指定哪些元素应用样式。
- 在 JavaScript 中指定要查找哪个元素。

例如：

```js
document.querySelector("#submit");
document.querySelector('input[name="email"]');
document.querySelector('[data-testid="login-button"]');
document.querySelector(".confirm-btn");
```

`document.querySelector(...)` 会返回第一个匹配选择器的元素。

### 6.1 常见选择器

按 id 选择：

```js
document.querySelector("#submit");
```

意思是找 `id="submit"` 的元素。

按标签 + 属性选择：

```js
document.querySelector('input[name="email"]');
```

意思是找 `name="email"` 的 `input`。

按属性选择：

```js
document.querySelector('[data-testid="login-button"]');
```

意思是找 `data-testid="login-button"` 的元素。

按 class 选择：

```js
document.querySelector(".confirm-btn");
```

意思是找 class 中包含 `confirm-btn` 的元素。

组合选择器可以理解成不断加条件，也就是“并且”。

例如：

```css
div.active
```

意思是：

**是 `div`，并且 class 里有 `active`。**

### 6.2 `nth-child`

例如：

```js
document.querySelector("div > button:nth-child(2)");
```

可以拆成：

- `div`：先看 `div`。
- `>`：只看直接子元素。
- `button:nth-child(2)`：这个按钮必须是父元素的第 2 个子元素。

关键点：

`**nth-child(2)` 不是“第二个 button”，而是“这个元素在父元素所有孩子里排第 2”。**

例如：

```html
<div>
  <span>1</span>
  <button>2</button>
  <button>3</button>
</div>
```

这里 `button:nth-child(2)` 能选中第一个按钮，因为它是父元素的第 2 个子元素。

### 6.3 `li:nth-child(9)`

```css
li: nth-child(9);
```

意思是：

**是 `li`，并且是父元素的第 9 个子元素。**

这里的 `:` 是 CSS 伪类语法，可以先理解成“附加筛选条件”。

常见例子：

- `li:first-child`：是 `li`，并且是第一个子元素。
- `li:last-child`：是 `li`，并且是最后一个子元素。
- `li:nth-child(9)`：是 `li`，并且是第 9 个子元素。

## 7. JavaScript 操作页面基础

JavaScript 经常通过 DOM API 找元素、绑定事件、修改页面内容。

### 7.1 点击计数器例子

```js
let value = 0;

document.getElementById("add").onclick = () => {
  value++;
  document.getElementById("value").innerHTML = value;
};
```

这段代码的作用是：

**每点击一次 id 为 `add` 的按钮，就让计数加 1，并把新数字显示到 id 为 `value` 的元素里。**

逐行理解：

- `let value = 0;`：定义一个变量，初始值是 `0`。
- `document.getElementById("add")`：找到 id 是 `add` 的元素。
- `.onclick = ...`：给这个元素绑定点击事件。
- `value++;`：让 `value` 自增 1，相当于 `value = value + 1`。
- `document.getElementById("value").innerHTML = value;`：把页面上 id 为 `value` 的元素内容改成当前数字。

### 7.2 `.onclick = () => { ... }` 怎么读

```js
document.getElementById("add").onclick = () => {
  value++;
};
```

可以拆成三块：

- `document.getElementById("add")`：谁，找到 id 为 `add` 的元素。
- `.onclick`：什么时候，表示点击这个元素时。
- `= () => { ... }`：做什么，把右边这个函数绑定到点击事件上。

这里的 `=` 和 `=>` 分工不同：

- `=`：赋值，把右边的函数交给左边的 `onclick`。
- `=>`：定义一个箭头函数，把一组动作打包起来，等点击时再执行。

所以这句可以翻译成：

**把“点击后要执行的动作”赋值给这个按钮的点击事件。**

### 7.3 为什么要包成函数

下面这种写法不合适：

```js
document.getElementById("add").onclick = value++;
```

因为 `value++` 会在代码加载时立刻执行，而不是等用户点击时执行。

用函数包起来：

```js
document.getElementById("add").onclick = () => {
  value++;
};
```

意思是：

**先别执行，等点击发生时再执行。**

### 7.4 箭头函数和传统函数

箭头函数：

```js
() => {
  value++;
};
```

传统函数：

```js
function() {
  value++;
}
```

在这个点击事件例子里，可以先把它们理解成同一类东西：

**都是一段将来被调用的动作。**

箭头函数只是更现代、更短的写法。

### 7.5 `innerHTML`

```js
document.getElementById("value").innerHTML = value;
```

意思是：

**把某个元素内部的 HTML 内容替换成 `value` 当前的值。**

如果只是显示纯文本，后面也会常见：

```js
element.textContent = value;
```

`textContent` 更偏纯文本，`innerHTML` 可以解析 HTML 字符串。

### 7.6 `fetch`

`fetch` 是 JavaScript 里发网络请求的 API。

可以先理解成：

**去某个 URL 拿数据或文件。**

例如：

```js
fetch("https://api.example.com/data");
```

意思是向这个地址发起请求。

常见用法：

```js
fetch("https://api.example.com/data")
  .then(response => response.json())
  .then(data => {
    console.log(data);
  });
```

可以读成：

1. 去接口拿数据。
2. 把响应解析成 JSON。
3. 使用解析后的数据。

如果看到类似：

```js
fetch("https://unpkg.com/react@18.2.0/...");
```

通常是在从远程 CDN 获取某个库文件。

## 8. CSS 规则

CSS 规则的基本格式：

```css
选择器 {
  属性: 值;
}
```

例如：

```css
h1 {
  color: red;
}
```

意思是：

**所有 `h1` 元素，文字颜色设为红色。**

大括号 `{}` 表示这个选择器选中的元素要应用哪些样式规则。

规则块里的基本格式：

```css
属性: 值;
```

符号含义：

- `:`：分隔属性和属性值。
- `;`：表示这一条规则结束。
- `{}`：表示这一组规则属于当前选择器。

### 8.1 `!important`

`!important` 是 CSS 里的优先级提升标记。

可以先理解成：

**这条样式很重要，和别的样式冲突时，优先用我。**

例子：

```css
.title {
  color: blue;
}

.title {
  color: red !important;
}
```

最终通常会显示红色。

但 `!important` 不是绝对无敌。如果两条规则都写了 `!important`，浏览器还会继续比较：

- 谁的选择器更具体。
- 谁写得更靠后。

### 8.2 `visibility: hidden` 和 `display: none`

`visibility: hidden`：

**看不见，但位置还在。**

`display: none`：

**直接从布局里消失，不占位置。**

### 8.3 DevTools 注入的隐藏样式

例如：

```html
<style id="__web-inspector-hide-shortcut-style__">
  .__web-inspector-hide-shortcut__,
  .__web-inspector-hide-shortcut__ * {
    visibility: hidden !important;
  }
</style>
```

可以理解成：

**把 class 为 `__web-inspector-hide-shortcut__` 的元素，以及它里面所有子元素，都隐藏掉。**

其中：

- `,` 表示“或者”。
- `.` 表示按 class 选择。
- `*` 表示所有元素。
- `!important` 表示提高优先级。

## 9. CSS 条件规则

CSS 中带 `@` 的语法叫 at-rule，可以先理解成：

**CSS 里的特殊指令。**

普通选择器负责“选谁”，`@` 规则负责“在什么条件或场景下应用 CSS”。

常见 at-rule：

- `@media`
- `@supports`
- `@container`
- `@keyframes`
- `@font-face`
- `@import`

### 9.1 `@media`

`@media` 根据整个浏览器视口判断。

```css
@media (max-width: 500px) {
  .box1 {
    background: yellow;
  }
}
```

意思是：

**当浏览器窗口宽度小于等于 `500px` 时，让 `.box1` 背景变黄。**

### 9.2 `@supports`

`@supports` 用来检测浏览器是否支持某个 CSS 能力。

```css
@supports (container-type: size) {
  .box1 {
    background: yellow;
  }
}
```

意思是：

**如果浏览器支持 `container-type: size`，就应用里面的规则。**

### 9.3 `@container`

`@container` 是容器查询。

它和 `@media` 很像，但判断对象不同：

- `@media`：根据整个浏览器窗口判断。
- `@container`：根据某个容器元素自己的尺寸判断。

示例：

```css
body {
  padding: 20px;
  container-type: size;
  container-name: body;
}

@container body (max-width: 500px) {
  .box1 {
    background: red;
  }
}
```

这段可以翻译成：

**当名叫 `body` 的容器宽度不超过 `500px` 时，让它里面的 `.box1` 背景变红。**

这里 `@container body (...)` 里的 `body` 不是标签选择器，而是容器名字：

```css
container-name: body;
```

### 9.4 `container-type`

```css
container-type: size;
```

它不是为了改变视觉效果，而是一个“开关”：

**让这个元素成为可被 `@container` 查询尺寸的容器。**

为什么要显式声明？

因为容器查询会牵涉布局计算。如果所有元素都天然可查，浏览器要维护的依赖关系会很复杂，还可能出现循环：

```text
子元素样式依赖父容器尺寸
父容器尺寸又被子元素撑开
子元素样式变化后又改变父容器尺寸
```

所以 CSS 要你显式写：

```css
container-type: size;
```

这等于告诉浏览器：

**我确认这个元素要当查询容器，你可以为它建立容器查询相关的计算规则。**

实际开发里更常见的是：

```css
container-type: inline-size;
```

因为多数响应式布局主要关心横向宽度。

### 9.5 条件嵌套

例如：

```css
@supports (container-type: size) {
  @media (max-width: 500px) {
    .box1 {
      background-color: yellow;
    }
  }
}
```

可以读成：

**浏览器支持 `container-type: size`，并且浏览器窗口宽度小于等于 `500px` 时，让 `.box1` 背景变黄。**

这是“并且”的关系。

### 9.6 CSS 注释

```css
/* 这里是注释 */
```

被 `/* ... */` 包起来的内容不会执行，浏览器会忽略。

## 10. XPath

`XPath` 是一种用来在 XML 或 HTML 文档里定位节点的语法。

如果已经理解 CSS 选择器，可以先把它理解成：

**另一套查找页面元素的写法。**

例如有这段 HTML：

```html
<div id="app">
  <button>提交</button>
</div>
```

CSS 选择器写法：

```js
document.querySelector("#app button");
```

XPath 写法：

```text
//*[@id="app"]//button
```

XPath 常见于：

- 自动化测试。
- 爬虫。
- Selenium 这类浏览器自动化。
- 某些旧工具链。

XPath 按文本找元素很常见：

```text
//button[text()="提交"]
```

意思是：

**找文字刚好是“提交”的 `button`。**

对浏览器插件开发来说，通常优先用 CSS 选择器，因为 `querySelector` / `querySelectorAll` 原生支持。

## 11. 选择元素的实用技巧

不要一上来就写：

```js
document.querySelector("button");
```

因为这通常只会取到页面里的第一个按钮，不稳定。

更推荐的优先级：

1. `id`
2. 稳定的 `name`
3. 稳定的 `data-*`
4. 稳定的 `class`
5. 结构关系
6. `nth-child`

先找容器，再在容器里找：

```js
const panel = document.querySelector("#login-panel");
const btn = panel.querySelector('button[type="submit"]');
```

DevTools 小技巧：

```js
$0;
```

在 `Elements` 面板里选中一个元素后，控制台里的 `$0` 就是当前选中的那个元素。

可以直接试：

```js
$0.textContent;
$0.className;
$0.getAttribute("name");
```

## 12. 网页入口与 JS 执行顺序

传统网页的入口通常是 HTML 文件，常见名字是：

```text
index.html
```

浏览器打开网页时，通常先读取 HTML，再根据 HTML 里的标签加载 CSS、JS、图片等资源。

### 12.1 JS 文件怎么被执行

JS 文件一般是被 HTML 里的 `<script>` 标签引入后才会执行。

例如：

```html
<script src="index.js"></script>
```

意思是：

**浏览器读到这里时，去加载并执行 `index.js`。**

如果一个 JS 文件没有被 HTML 引入，也没有被其他 JS `import`，浏览器通常不会主动执行它。

### 12.2 多个 `<script>` 的执行顺序

```html
<script src="config.js"></script>
<script src="index.js"></script>
```

默认情况下，浏览器会按 HTML 中出现的顺序执行。

也就是先执行 `config.js`，再执行 `index.js`。

### 12.3 现代前端项目的入口

现代项目常见写法是：

```html
<script type="module" src="./src/main.js"></script>
```

这表示 HTML 只引入一个总入口 JS。

然后 `main.js` 内部再通过 `import` 引入其他模块：

```js
import { initializeApp } from "./app.js";
import { setupCounter } from "./counter.js";

initializeApp();
setupCounter();
```

所以可以先记：

**HTML 是网页入口，`script` 点名 JS，JS 再通过 `import` 组织更多文件。**

### 12.4 浏览器扩展里的入口

浏览器扩展不只看 `index.html`，还要看：

```text
manifest.json
```

扩展里的入口由 `manifest.json` 声明，例如：

- 哪个文件是后台脚本。
- 哪个 HTML 是侧边栏或 popup。
- 哪些 content script 注入到哪些网页。

所以普通网页先找 `index.html`，浏览器扩展先看 `manifest.json`。
