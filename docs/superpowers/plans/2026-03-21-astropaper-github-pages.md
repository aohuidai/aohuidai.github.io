# AstroPaper GitHub Pages Blog Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bootstrap an `AstroPaper` blog in this repository, configure it for a `<username>.github.io` GitHub Pages user site, and leave the user with a fast Markdown-based publishing workflow.

**Architecture:** Use the upstream `satnaing/astro-paper` Astro template as the site foundation, keep the theme mostly intact, and limit customization to site metadata, sample content, and deployment wiring. Deploy with the current Astro-recommended GitHub Pages Actions workflow so the repository builds static files automatically on push.

**Tech Stack:** Astro, AstroPaper, TypeScript, Markdown content collections, GitHub Pages, GitHub Actions, npm

### Notes on bootstrapping

Because this repository root is not empty, `create-astro` will create a nested directory if run directly here. Scaffold the theme into a temporary directory first, then copy the generated project files into the repository root while preserving `.git/`, `docs/`, and any existing local-only folders that should remain untouched.
